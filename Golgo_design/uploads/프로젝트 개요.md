# 프로젝트 개요: AI-Driven 개인 맞춤형 리밸런싱 어드바이저 (Golgo (고르고))

## 1. 프로젝트 목적

- **증권사 Open API**와 LLM(대규모 언어 모델)을 결합하여 개인의 투자 성향에 최적화된 종목 분석 및 리밸런싱 가이드를 제공함.
    
- 단순 수치 나열이 아닌, **50년 경력의 베테랑 분석가 페르소나**를 통해 깊이 있는 거시경제 및 재무 통찰력을 사용자에게 전달.
    
- 수동 매매로 시작하여 향후 자동 리밸런싱 시스템으로 확장 가능한 유연한 아키텍처 구축.
- 주식 초보자가 사용하기 쉽게 사용자 친화적이고 직관적인 UI 및 서비스 이용 가능하도록 구축

## 2. 핵심 기능 (MVP)

- **자산 현황 실시간 동기화:** 한국투자증권/미래에셋 API를 통한 보유 종목 및 비중 자동 로드.
    
- **사용자 투자 전략 설정:** 적립식 지수 투자, 공격적 섹터 투자 등 개인화된 전략 저장.
    
- **Deep-Inference 리포트 생성:**
    
    - Gemini/GPT/Claude 기반의 7단계 심층 분석 (비즈니스 모델, 산업 구조, 재무, 밸류에이션, 실적, 거시경제, 촉매/리스크).
        
- **리밸런싱 가이드 제공:** 목표 비중과 현재 비중의 괴리율을 계산하여 매수/매도 필요 수량 산출.
    
- **대시보드 요약:** 핵심 투자 논리(Investment Thesis) 및 리스크 체크리스트 시각화.
    
    

## 3. AI 분석 프레임워크 (7단계 페르소나 로직)

분석 엔진은 다음 구조에 따라 `System Prompt`가 고정됨:

1. **기업 개요 및 수익 모델:** 실제 돈을 어디서 버는가?
    
2. **산업 및 경쟁 구조:** Moat(경제적 해자)와 사이클 위치 분석.
    
3. **재무 분석:** 3~5년 데이터 기반 FCF, ROIC, 실적의 질 평가.
    
4. **밸류에이션:** PER/PBR/PSR 및 DCF 관점의 가치 평가.
    
5. **실적 및 컨콜 분석:** 가이던스 변화 및 경영진 멘트 요약.
    
6. **거시경제 및 정책:** 금리, 환율, 정부 규제 영향도 분석.
    
7. **촉매와 리스크:** "내가 틀린다면?"을 포함한 자기반박 및 시나리오 분석.
    

## 4. 시스템 아키텍처 (Workflow)

1. **Data Fetching:** 증권사 API로 잔고 스캔 $\rightarrow$ 외부 뉴스/금융 데이터 크롤링.
    
2. **Prompt Construction:** `{{사용자 성향}}` + `{{현재 포트폴리오}}` + `{{시장 데이터}}` 결합.
    
3. **AI Inference:** LLM이 전문 분석가 페르소나로 분석 수행.
    
4. **Parsing & Mapping:** LLM의 답변을 JSON으로 파싱하여 대시보드 UI 매핑.
    
5. **User Action:** 사용자가 리포트 확인 후 직접 매매 수행 (추후 자동 주문 모듈 확장).
    

## 5. 개발 로드맵

- **Phase 1 (준비):** 증권사 API 연동(모의투자) 및 기본 잔고 조회 기능 구현.
    
- **Phase 2 (지능화):** Gemini API 연동 및 7단계 분석 프롬프트 튜닝.
    
- **Phase 3 (최적화):** 리밸런싱 수량 계산 로직 알고리즘화 및 UI 시각화.
    
- **Phase 4 (고도화):** 매수/매도 주문 API 연결 및 자동화 조건 설정 기능 추가.

## 6. 주의사항
- 각 API는 엔드포인트, Request Body(JSON), Response Body(JSON) 예시를 포함해야 합니다. 할루시네이션을 방지하기 위해 LLM의 응답은 엄격한 JSON 구조로 제한해야 합니다.
- 보안 및 API 키 관리: 사용자의 증권사 Access Token 및 계좌 비밀번호는 반드시 강력하게 암호화되어 관리되어야 합니다. 백엔드 기획 시보안 솔루션을 포함해야 합니다.
-  LLM 응답의 예측 가능성: LLM이 갑자기 텍스트로 답하면 시스템이 터집니다. 반드시 Structured Outputs(구조화된 출력 기능)이나 LangChain의 Pydantic Parser를 사용하여 {"TSLA": 0.4, "AAPL": 0.6}과 같은 엄격한 형식만 반환하도록 제한해야 합니다.
- 증권사 API 제한 사항: 국내 증권사 API는 대부분 1초당 요청 제한(Rate Limit)이 걸려 있고, 모바일 앱 환경에서 웹소켓이나 REST 통신 시 토큰 갱신 주기 처리가 까다롭습니다. 이 처리를 담당할 Queue(예: Celery, Redis) 배치 시스템 기획이 필수적입니다.
