# 로그인 후 온보딩 상태 기반 이동 설계

## 목표

아이디/비밀번호 로그인과 소셜 로그인 모두에서 사용자 계정의 `onboardingCompleted` 값에 따라 다음 화면을 결정한다.

- `true`: 현재 대시보드 진입점인 `/`로 이동
- `false`: 각 로그인 흐름의 기존 설정 화면으로 이동

## 범위

- 아이디/비밀번호 로그인 성공 후 이동 경로 분기
- 소셜 로그인 콜백 성공 후 이동 경로 분기
- 두 로그인 방식의 분기 동작에 대한 프론트엔드 테스트

대시보드 화면 자체의 구현과 인증된 사용자의 직접 URL 접근 제한은 포함하지 않는다.

## 설계

로그인 API와 `/me` 응답에서 이미 제공하는 `AuthUser.onboardingCompleted` 값을 사용한다. 새로운 API나 전역 상태는 추가하지 않는다.

아이디/비밀번호 로그인은 `loginWithPassword` 반환 사용자를 확인해 완료 사용자는 `/`, 미완료 사용자는 `/onboarding`으로 이동한다. 소셜 로그인 콜백은 `loadUser` 반환 사용자를 확인해 완료 사용자는 `/`, 미완료 사용자는 기존 흐름인 `/nickname`으로 이동한다.

분기는 각 로그인 페이지에 직접 두어 현재 두 호출 지점만 수정한다. 공통 함수나 전역 Route Guard는 이번 범위에서 추가하지 않는다.

## 오류 처리

로그인 또는 사용자 조회 실패 시 현재 오류 처리와 화면을 유지한다. `onboardingCompleted`는 응답 타입상 필수 boolean이므로 별도 fallback은 두지 않는다.

## 검증

- 아이디/비밀번호 로그인: 완료 사용자가 `/`로 이동
- 아이디/비밀번호 로그인: 미완료 사용자가 `/onboarding`으로 이동
- 소셜 로그인: 완료 사용자가 `/`로 이동
- 소셜 로그인: 미완료 사용자가 `/nickname`으로 이동
- 프론트엔드 lint 및 build 통과
