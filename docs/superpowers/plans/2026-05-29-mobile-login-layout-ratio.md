# 모바일 로그인 레이아웃 비율 조정 계획

## 목표

모바일 화면을 기준으로 SSO 로그인 화면의 세로/가로 비율을 정리한다. 작은 모바일 화면에서도 핵심 문구, 체크포인트, 로그인 버튼이 과도하게 늘어나거나 잘리지 않고 한 화면 안에서 자연스럽게 배치되도록 한다.

## 현재 상태

- `LoginPage`의 앱 프레임이 `min-h-[720px]`로 고정되어 있어 667px 높이 계열 모바일에서 화면이 과하게 길다.
- 내부 콘텐츠 영역이 `justify-center`와 큰 여백(`pt-12`, `pb-10`)을 사용해 버튼 영역이 아래로 밀린다.
- 모바일 기준 버튼 높이와 이미지 비율은 보정됐지만, 전체 레이아웃 높이와 패딩은 아직 데스크톱 프레임에 가깝다.
- `frontend/src/assets/NAVER_login.png`, `frontend/src/assets/google_login.png`는 현재 작업트리에 사용자 변경으로 남아 있으므로 이번 작업에서 함께 수정정

## 수정 범위

- 수정: `frontend/src/features/auth/pages/LoginPage.tsx`
- 필요 시 수정: `frontend/src/features/auth/components/SocialLoginButton.tsx`
- 수정하지 않음: 로그인 버튼 이미지 파일 2개

## 구현 방침

1. 모바일 앱 프레임을 실제 viewport 기준으로 조정한다.
   - 기존 `min-h-[720px]`를 모바일에서는 `min-h-[calc(100svh-2rem)]`에 가깝게 바꾼다.
   - 데스크톱에서는 기존처럼 충분한 높이를 유지한다.

2. 프레임 폭과 패딩을 모바일 기준으로 줄인다.
   - 모바일 최대 폭은 `390px`를 유지하되, 화면 폭이 작을 때 좌우 패딩이 과하지 않도록 `px-5` 수준으로 낮춘다.
   - 데스크톱에서는 기존 `px-7` 수준을 유지한다.

3. 콘텐츠 세로 비율을 상단 브랜드/중앙 메시지/하단 버튼으로 나눈다.
   - 로고와 헤드라인 사이 여백을 줄인다.
   - 체크포인트 카드 높이와 패딩을 줄여 3개 카드가 모바일에서 부담스럽지 않게 한다.
   - 버튼 그룹은 하단에 안정적으로 붙이되, 약관 문구까지 보이도록 한다.

4. 텍스트 크기를 모바일 우선으로 조정한다.
   - 헤드라인은 모바일에서 `30px` 전후로 줄이고, `sm` 이상에서 현재 크기로 키운다.
   - 체크포인트 텍스트는 줄바꿈이 생겨도 카드 높이를 크게 밀지 않게 한다.

5. 버튼 이미지 비율은 레이아웃 비율에 맞게 조정정

## 검증 계획

1. 정적 검증

```powershell
npm run lint
npm run build
```

2. 로컬 라우트 확인

```powershell
npm run dev -- --host 127.0.0.1
```

확인 URL:

```text
http://127.0.0.1:3000/login
```

3. 모바일 기준 확인 항목

- 390x844 기준에서 전체 프레임이 화면에 자연스럽게 들어온다.
- 375x667 기준에서 버튼과 약관 문구가 과하게 밀리지 않는다.
- 네이버/구글 버튼 이미지가 잘리지 않는다.
- 체크포인트 카드 3개가 한 줄에서 깨지지 않는다.
- 텍스트가 버튼이나 카드 경계를 넘지 않는다.
