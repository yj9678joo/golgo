 4. 개발 기획 시 반드시 고려해야 할 핵심 체크리스트
1. 보안 및 API 키 관리: 사용자의 증권사 Access Token 및 계좌 비밀번호는 반드시 강력하게 암호화되어 관리되어야 합니다. 백엔드 기획 시 AWS Secrets Manager나 Vault 같은 보안 솔루션을 포함해야 합니다.
2. LLM 응답의 예측 가능성: LLM이 갑자기 텍스트로 답하면 시스템이 터집니다. 반드시 Structured Outputs(구조화된 출력 기능)이나 LangChain의 Pydantic Parser를 사용하여 {"TSLA": 0.4, "AAPL": 0.6}과 같은 엄격한 형식만 반환하도록 제한해야 합니다.
3. 증권사 API 제한 사항: 국내 증권사 API는 대부분 1초당 요청 제한(Rate Limit)이 걸려 있고, 모바일 앱 환경에서 웹소켓이나 REST 통신 시 토큰 갱신 주기 처리가 까다롭습니다. 이 처리를 담당할 Queue(예: Celery, Redis) 배치 시스템 기획이 필수적입니다.
