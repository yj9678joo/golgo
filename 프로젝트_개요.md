# 프로젝트 개요: AI-Driven 개인 맞춤형 리밸런싱 어드바이저 (Golgo (고르고))

## 1. 프로젝트 목적

- **증권사 Open API**와 LLM(대규모 언어 모델)을 결합하여 개인의 투자 성향에 최적화된 종목 분석 및 리밸런싱 가이드를 제공함.
- 단순 수치 나열이 아닌, **50년 경력의 베테랑 분석가 페르소나**를 통해 깊이 있는 거시경제 및 재무 통찰력을 사용자에게 전달.
- 수동 매매로 시작하여 향후 자동 리밸런싱 시스템으로 확장 가능한 유연한 아키텍처 구축.
- 주식 초보자가 사용하기 쉽게 사용자 친화적이고 직관적인 UI 및 서비스 이용 가능하도록 구축

## 2. 핵심 기능 (MVP)

- **자산 현황 실시간 동기화:** 한국투자증권/미래에셋 API를 통한 보유 종목 및 비중 자동 로드.
- **사용자 투자 전략 설정:** 적립식 지수 투자, 공격적 섹터 투자 등 개인화된 전략 저장.
- **Deep-Inference 리포트 생성:**
  - Gemini/GPT/Claude 기반의 7단계 심층 분석 (비즈니스 모델, 산업 구조, 재무, 밸류에이션, 실적, 거시경제, 촉매/리스크).
- **리밸런싱 가이드 제공:** 목표 비중과 현재 비중의 괴리율을 계산하여 매수/매도 필요 수량 산출.
- **대시보드 요약:** 핵심 투자 논리(Investment Thesis) 및 리스크 체크리스트 시각화.

## 3. AI 분석 프레임워크 (7단계 페르소나 로직)

분석 엔진은 다음 구조에 따라 `System Prompt`가 고정됨:

1. **기업 개요 및 수익 모델:** 실제 돈을 어디서 버는가?
2. **산업 및 경쟁 구조:** Moat(경제적 해자)와 사이클 위치 분석.
3. **재무 분석:** 3~5년 데이터 기반 FCF, ROIC, 실적의 질 평가.
4. 밸류에이션 분석 (Valuation Metrics & Rationale)
   - **A. TTM P/E (과거 12개월 실적 기준) 검증 및 착시 분석**
     - **공식:** $TTM\ P/E = \frac{\text{현재 주가}}{\text{과거 12개월 주당순이익}(EPS)}$
     - **분석 논리:** 현재 수치가 역사적 고평가 영역(예: 삼성전자 약 41배 수준)에 도달했을 때, 이것이 주가의 절대적 버블인지 아니면 '사이클 저점에서 발생한 일시적 착시(이익 부진)'인지를 냉정하게 판정한다.
     - **질적 평가:** 이미 지나간 과거 실적에 시장의 미래 기대치(AI 인프라 호황 등)가 얼마나 선반영되어 있는지 그 괴리율을 추적한다.
   - **B. FY2 P/E (내년/차차기년도 예상 실적 기준) 기반의 포워드 분석**
     - **공식:** $FY2\ P/E = \frac{\text{현재 주가}}{\text{차차기년도 예상 주당순이익}(Expected\ EPS)}$
     - **분석 논리:** 사이클 초입에서 급증할 미래 이익(예: 2026~2027년 AI 메모리 수요 폭증 분)을 분모에 대입하여 실제 포워드 밸류에이션(예: 삼성전자 기준 5~7배 수준으로 하락하는 현상)을 산출한다.
     - **스트레스 테스트 (함정 검증):** 분모인 '예상 이익'의 하방 리스크를 반드시 점검한다. 만약 AI 수요 둔화나 매크로 충격으로 실제 이익이 전망치의 50%에 그칠 경우, 현재의 FY2 P/E가 어떻게 왜곡(예: 5배 $\rightarrow$ 10배)될 수 있는지 최악의 시나리오를 가정한 주가 하방 지지선을 계산한다.
     - 과거 실적(TTM 41배)의 고평가 착시를 내년/차차기년도 예상 이익(FY2 5~7배)으로 보정하되, 이익 전망치의 하방 리스크(스트레스 테스트)를 반드시 병행함.

   - **C. PEG (주가수익성장비율) 기반의 성장성 대비 가격 평가**
     - **공식:** $PEG = \frac{FY2\ P/E}{\text{예상 이익 성장률}(\%)}$
     - **분석 논리:** 같은 P/E 멀티플을 부여받더라도 이익의 성장 속도가 다르면 가치가 다르다는 점을 명시한다. 성장 속도 대비 현재 주가가 합리적인지 판정한다.
     - **해석 가이드라인:**
       - **$PEG < 1$ (적극 매수 구간):** 예상되는 이익 성장 속도에 비해 주가가 매우 저평가 상태(예: SK하이닉스 0.18, 삼성전자 0.28 수준). 시장이 성장의 강도를 과소평가하고 있음을 시사.
       - **$PEG \ 1 \sim 2$ (적정 가치 구간):** 성장에 걸맞은 제값을 쥐고 있는 상태.
       - **$PEG > 2$ (과열/매도 검토 구간):** 성장 속도에 비해 시장의 내러티브와 낙관론이 과도하게 반영된 상태.
     - 공식을 통해, 성장성 대비 주가가 제값(PEG 1~2)인지, 극단적 저평가(PEG 1 미만)인지를 판정함.
   - **D. PBR (재무 상태 및 청산가치 기반 밸류에이션)**
     - **공식:** $PBR = \frac{\text{주가}}{\text{주당순자산가치}(BPS)}$ / $\text{BPS} = \frac{\text{총자산} - \text{총부채}}{\text{발행 주식수}}$
     - **분석 논리:** PER이 '지금 이 회사가 얼마를 버는가(수익성)'를 본다면, PBR은 '이 회사가 지금 당장 망해서 문을 닫으면 내 손에 얼마가 떨어지는가(재무 안전성/청산가치)'를 측정한다.
     - **해석 및 함정 검증:**
       - **$PBR < 1$ (청산가치 이하):** 이론적으로 회사의 장부상 가치보다 주가가 낮으므로 극단적 저평가 영역이다.
       - **PBR:** 장치 산업(반도체 등)에서 PBR이 낮다고 무조건 사면 안 된다. 불황기에 접어들어 공장 가치(자산)는 그대로인데 가동률이 떨어져 적자가 누적되면, 순자산 자체가 깎여 나가면서 PBR이 도로 높아지는 '밸류 트랩(Value Trap)'이 발생할 수 있다. 반대로 호황기 초입에는 PBR이 다소 높아 보여도 과감한 배팅이 필요할 수 있음을 분석에 반영한다.
   - **E. PSR (매출 성장성 기반 밸류에이션)**
     - **공식:** $PSR = \frac{\text{주가}}{\text{주당 매출액}(SPS)}$
     - **분석 논리:** 아직 대규모 투자가 진행 중이라 이익(Net Income)은 적자이거나 미미하지만, 시장 점유율을 무섭게 장악하며 매출(Sales)이 폭발하는 **초기 고성장 기업/섹터**를 평가하는 핵심 지표다.
     - **해석 및 함정 검증:**
       - **낮은 PSR:** 기업의 외형 성장 잠재력에 비해 주가가 지나치게 소외되어 있음을 뜻한다.
       - **PSR 맹점:** 매출만 늘고 마진(영업이익률)이 전혀 개선되지 않는 '외화내빈형' 기업을 걸러내야 한다. PSR을 볼 때는 반드시 "매출 성장이 추후 영업이익률(OPM) 리바운드로 이어질 수 있는 구조적 해자(Moat)가 있는가?"를 세트로 묶어서 추론한다.

5. **실적 및 컨콜 분석:** 가이던스 변화 및 경영진 멘트 요약.
6. **거시경제 및 정책:** 금리, 환율, 정부 규제 영향도 분석.
7. **촉매와 리스크:** "내가 틀린다면?"을 포함한 자기반박 및 시나리오 분석.

```md
[System Valuation Logic]
LLM은 대상 종목을 평가할 때 반드시 다음 3가지 차원을 교차 검증(Cross-Check)해야 합니다.

1. 수익성 차원 (PER, PEG): 미래에 벌어들일 성장 속도에 비해 싼가?
2. 안정성 차원 (PBR): 업황이 최악으로 치닫더라도 장부상 자산이 주가를 방어해 줄 수 있는가?
3. 성장성 차원 (PSR): 이익이 찍히기 전이라도, 매출 성장을 통한 시장 지배력 확대가 일어나고 있는가?

이 3가지 지표가 서로 모순될 때(예: PBR은 0.5로 싸지만 PSR이 높아 매출이 꺾이는 경우 등)는
그 원인을 거시경제 환경(금리, 전방 산업 수요)과 연결하여 '경고 문구'와 함께 자산 배분 비중 축소를 제안하십시오.
```

## 4. 시스템 아키텍처 (Workflow)

1. **Data Fetching:** 증권사 API로 잔고 스캔 $\rightarrow$ 외부 뉴스/금융 데이터 크롤링.
2. **Prompt Construction:** `{{사용자 성향}}` + `{{현재 포트폴리오}}` + `{{시장 데이터}}` 결합.
3. **AI Inference:** LLM이 전문 분석가 페르소나로 분석 수행.
4. **Parsing & Mapping:** LLM의 답변을 JSON으로 파싱하여 대시보드 UI 매핑.
5. **User Action:** 사용자가 리포트 확인 후 직접 매매 수행 (추후 자동 주문 모듈 확장).

## 5. 개발 로드맵

- **Phase 1 (준비):** 증권사 API 연동(모의투자) 및 기본 잔고 조회 기능 구현.
- **Phase 2 (지능화):** Gemini API 연동 및 7단계 분석 프롬프트 튜닝.
- **Phase 3 (최적화):** 리밸런싱 수량 계산 로직 알고리즘화 및 UI 시각화.
- **Phase 4 (고도화):** 매수/매도 주문 API 연결 및 자동화 조건 설정 기능 추가.
- 원하는 종목 추가해서 분석해주는 기능 추가

## 6. 주의사항

- 각 API는 엔드포인트, Request Body(JSON), Response Body(JSON) 예시를 포함해야 합니다. 할루시네이션을 방지하기 위해 LLM의 응답은 엄격한 JSON 구조로 제한해야 합니다.
- 보안 및 API 키 관리: 사용자의 증권사 Access Token 및 계좌 비밀번호는 반드시 강력하게 암호화되어 관리되어야 합니다. 백엔드 기획 시보안 솔루션을 포함해야 합니다.
- LLM 응답의 예측 가능성: LLM이 갑자기 텍스트로 답하면 시스템이 터집니다. 반드시 Structured Outputs(구조화된 출력 기능)이나 LangChain의 Pydantic Parser를 사용하여 {"TSLA": 0.4, "AAPL": 0.6}과 같은 엄격한 형식만 반환하도록 제한해야 합니다.
- 증권사 API 제한 사항: 국내 증권사 API는 대부분 1초당 요청 제한(Rate Limit)이 걸려 있고, 모바일 앱 환경에서 웹소켓이나 REST 통신 시 토큰 갱신 주기 처리가 까다롭습니다. 이 처리를 담당할 Queue(예: Celery, Redis) 배치 시스템 기획이 필수적입니다.
